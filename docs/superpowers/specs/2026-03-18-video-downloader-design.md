## 1. 项目概述

### 1.1 项目目标

开发一款功能全面的桌面视频下载器应用，支持主流视频网站（YouTube、Bilibili等）和国内视频平台（优酷、爱奇艺、腾讯视频等），提供友好的用户界面和完善的下载管理功能。

### 1.2 目标用户

- 普通用户：需要下载视频进行离线观看
- 内容创作者：需要下载素材进行二次创作
- 教育工作者：需要下载教学视频资源

### 1.3 核心功能

1. **基础下载** - 输入视频URL，下载视频文件
2. **批量下载** - 支持同时下载多个视频或播放列表
3. **画质选择** - 支持选择不同清晰度（如1080p, 720p, 480p等）
4. **格式转换** - 支持转换为不同视频格式（MP4, MKV, AVI等）
5. **音频提取** - 仅下载音频（MP3, AAC等）
6. **字幕下载** - 下载视频字幕文件
7. **下载管理** - 暂停/继续/取消下载，查看下载进度

```
                            ↓

└─────────────────────────────────────────────────────────────┘
```

### 3.2 设计原则

- **分层设计**：清晰的三层架构，职责分离
- **MVC模式**：Model-View-Controller分离，便于维护和测试
- **异步处理**：下载任务在后台线程运行，不阻塞UI
- **依赖注入**：组件之间通过接口依赖，便于测试和扩展
- **线程安全**：使用QMutex保护共享资源，确保多线程安全
- **事件驱动**：使用PyQt信号槽机制，降低组件耦合度

### 3.3 线程安全设计

**共享资源保护：**
- 下载队列：使用QMutex保护，确保线程安全
- 配置文件：使用文件锁避免并发写入
- 日志系统：Python logging模块线程安全

**线程边界：**
- UI线程：仅处理界面更新和用户交互
- 工作线程：处理下载、解析等耗时操作
- 使用信号槽机制进行线程间通信

**并发控制：**
- 默认并发下载数：3个
- 可配置并发数：1-10个
- 下载队列持久化：应用重启后恢复未完成任务

## 5. 数据流

### 5.1 下载流程

```
用户输入URL → 点击下载
    ↓
[视频解析器] 解析URL，获取视频信息
    ↓
[下载管理器] 创建下载任务
    ↓
[下载任务] 开始下载（后台线程）
    ↓
[yt-dlp封装器] 调用yt-dlp下载
    ↓
[下载任务] 更新进度 → [UI] 更新进度条
    ↓
下载完成 → [下载任务] 保存文件
    ↓
[UI] 显示完成状态
```

**关键点：**
- 解析和下载在后台线程运行，不阻塞UI
- 进度实时更新到UI
- 下载失败时自动重试或提示用户

### 5.2 批量下载流程

```
用户输入多个URL或播放列表URL
    ↓
[视频解析器] 解析所有视频信息
    ↓
[下载管理器] 创建多个下载任务
    ↓
[下载队列] 按并发限制开始下载
    ↓
每个任务独立下载，互不影响
    ↓
全部完成后显示汇总信息
```

### 5.3 配置保存流程

```
用户修改设置（如下载路径）
    ↓
[配置管理器] 验证设置有效性
    ↓
保存到配置文件（JSON格式）
    ↓
应用重启后自动加载配置
```

## 6. 错误处理

### 6.1 错误码定义

```python
class ErrorCode:
    # 网络错误 (1000-1999)
    NETWORK_TIMEOUT = 1001
    NETWORK_DISCONNECTED = 1002
    PROXY_ERROR = 1003

    # 解析错误 (2000-2999)
    INVALID_URL = 2001
    VIDEO_DELETED = 2002
    VIDEO_PRIVATE = 2003
    VIDEO_REQUIRES_LOGIN = 2004

    # 下载错误 (3000-3999)
    DISK_FULL = 3001
    PERMISSION_DENIED = 3002
    DOWNLOAD_INTERRUPTED = 3003

    # 格式错误 (4000-4999)
    UNSUPPORTED_FORMAT = 4001
    CONVERSION_FAILED = 4002

    # 系统错误 (5000-5999)
    YTDLP_NOT_FOUND = 5001
    FFMPEG_NOT_FOUND = 5002
    DEPENDENCY_ERROR = 5003
```

### 6.2 错误分类

| 错误类型 | 示例 | 错误码 | 处理方式 |
|---------|------|--------|---------|
| **网络错误** | 连接超时、网络中断 | 1001-1009 | 自动重试（最多3次，指数退避） |
| **解析错误** | URL无效、视频已删除 | 2001-2009 | 提示用户，不重试 |
| **下载错误** | 磁盘空间不足、权限问题 | 3001-3009 | 提示用户，暂停任务 |
| **格式错误** | 不支持的格式 | 4001-4009 | 提示用户选择其他格式 |
| **系统错误** | yt-dlp未安装、FFmpeg缺失 | 5001-5009 | 提示安装依赖 |

### 6.3 重试策略

**网络错误重试：**
- 最大重试次数：3次
- 重试间隔：指数退避（1s, 2s, 4s）
- 重试条件：仅网络相关错误
- 超时时间：30秒

**下载错误处理：**
- 磁盘空间不足：暂停任务，提示用户清理
- 权限问题：提示用户修改保存路径
- 下载中断：支持断点续传（yt-dlp内置）

### 6.4 错误处理流程

```
错误发生
    ↓
记录错误日志（包含错误码）
    ↓
判断错误类型
    ↓
┌─────────────────────────────────────┐
│ 网络错误？ → 自动重试（最多3次）    │
│ 解析错误？ → 提示用户，不重试      │
│ 下载错误？ → 提示用户，暂停任务    │
│ 系统错误？ → 提示安装依赖          │
└─────────────────────────────────────┘
    ↓
更新UI显示错误信息（友好提示）
    ↓
用户可选择重试或取消
```

### 6.3 日志系统

**日志级别：**
- DEBUG：详细调试信息
- INFO：正常操作信息
- WARNING：警告信息（可恢复）
- ERROR：错误信息（需要处理）

**日志文件：**
- 位置：`logs/app.log`
- 轮转：按日期分割，保留最近7天
- 格式：时间戳 + 级别 + 消息

### 6.4 用户友好的错误提示

**错误提示设计原则：**
1. **简洁明了** - 用通俗语言描述错误
2. **提供解决方案** - 告诉用户如何解决
3. **避免技术术语** - 普通用户也能理解

**示例：**
- ❌ "yt-dlp subprocess returned non-zero exit status 1"
- ✅ "无法下载此视频，可能视频已删除或需要登录"

## 8. 部署方案

### 8.1 打包工具

使用 PyInstaller 打包成 Windows 可执行文件：

```bash
pyinstaller --onefile --windowed \
    --name "视频下载器" \
    --hidden-import=yt_dlp \
    --hidden-import=PyQt6 \
    --add-data "config.json;." \
    --add-data "logs;logs" \
    --upx-dir=upx \
    main.py
```

**打包参数说明：**
- `--onefile`：打包成单个exe文件
- `--windowed`：无控制台窗口
- `--hidden-import`：隐藏导入依赖模块
- `--add-data`：添加数据文件
- `--upx-dir`：使用UPX压缩减小体积

### 8.2 依赖管理

使用 `requirements.txt` 管理依赖：

**运行时依赖：**
```
PyQt6>=6.5.0
yt-dlp>=2024.0.0
```

**开发依赖：**
```
pyinstaller>=6.0.0
pytest>=7.0.0
pytest-qt>=4.0.0
pytest-mock>=3.0.0
black>=23.0.0
mypy>=1.0.0
flake8>=6.0.0
isort>=5.0.0
```

**FFmpeg依赖：**

FFmpeg 是视频格式转换和音频提取的必需组件，提供以下安装方式：

1. **自动安装（推荐）：**
   - 应用首次启动时检测 FFmpeg 是否存在
   - 如果不存在，提示用户下载安装
   - 提供一键下载脚本（从官方镜像下载）

2. **手动安装：**
   - 访问 [FFmpeg 官方下载页面](https://ffmpeg.org/download.html)
   - 下载 Windows 版本（推荐使用 BtbN 或 Gyan 构建）
   - 解压到任意目录，并添加到系统 PATH

3. **打包包含：**
   - 使用 PyInstaller 的 `--add-data` 参数包含 FFmpeg
   - 注意：会增加安装包体积约 50-100MB

4. **验证安装：**
   ```bash
   ffmpeg -version
   ```
   如果显示版本信息，则安装成功

```
├── logs\                # 日志目录

└── downloads\           # 默认下载目录
```

**安装包结构：**
```
视频下载器.exe
├── 内置资源
│   ├── yt-dlp (打包在exe内)
│   └── 配置模板
└── 运行时生成
    ├── %APPDATA%\视频下载器\config.json
    ├── %APPDATA%\视频下载器\logs\
    └── 用户选择的下载目录
```
